# Task: Sprint Planning

**Agent:** @sutherland-proj
**Trigger:** Um projeto foi aprovado na Betting Table (@singer-proj) ou início de um novo ciclo de sprint para projeto já em andamento.

## Entradas
- `project-brief-template.md` (Aprovado)
- E/ou Backlog pendente do projeto.

## Objetivo
1. Decompor o projeto maior em "Sprints" executáveis.
2. Definir o Backlog do Sprint e o Sprint Goal.
3. Alocar as tarefas para os Agentes/Squads corretos.

## O que o agente deve fazer
- Divida o escopo total no tempo do Appetite definido.
- Transforme os requisitos do brief em tasks / stories no formato `sprint-template.md`.
- Conduza o preenchimento e aprovação do `checklists/sprint-planning-checklist.md`.
- Se a tarefa for de outro squad (ex: copy para Vendas), prepare a chamada e o input usando `cross-squad-task-creation.md`.
- Priorize pelo risco e pelo que gera valor mais rápido (entregue software/valor funcionando cedo, não no fim do prazo inteiro).

## Saídas
- Arquivo de `sprint-template.md` preenchido.
- `sprint-planning-checklist.md` validado.
- (Opcional) Execução em batch da task `cross-squad-task-creation.md` para criar tarefas dependentes.
