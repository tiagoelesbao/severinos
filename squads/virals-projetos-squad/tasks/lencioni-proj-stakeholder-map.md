# Task: Stakeholder Mapping

**Agent:** @lencioni-proj
**Trigger:** Fase inicial de planejamento de projeto (durante/após Kickoff) de um cliente externo ou projeto interno de grande visibilidade.

## Entradas
- `project-brief-template.md` aprovado, especialmente a seção "Stakeholders".
- Informações de contatos fornecidos por @sutherland-proj no Kickoff.

## Objetivo
1. Classificar quem pode afundar, travar ou alavancar o projeto via matriz Poder/Interesse.
2. Determinar a cadência de comunicação correta para cada stakeholder (nem de mais, nem de menos).

## O que o agente deve fazer
- Mapeie todos os envolvidos em quatro caixas: "Gerenciar de perto", "Manter satisfeito", "Manter informado", "Monitorar".
- Monte um Plano de Comunicação contendo: O quê informar, Para Quem, Frequência e Canal.
- Especifique como a equipe de projeto se comunicará com o cliente (E-mail semanal? WhatsApp proativo? Call quinzenal?).

## Saídas
- Matriz de Stakeholders documentada como anexo/campo no ClickUp.
- Plano de Comunicações em formato pronto para ser integrado às rotinas do PM.
