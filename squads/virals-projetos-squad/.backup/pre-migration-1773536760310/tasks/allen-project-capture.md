# Task: Project Capture and Clarification

**Agent:** @allen-proj
**Trigger:** Diariamente ou sempre que um novo compromisso/urgência entra na vida do PM (Slack, Reuniões, E-mails não estruturados).

## Entradas
- Inbox bruto da equipe/PM.
- Notas mentais de compromissos ('Stuff').

## Objetivo
1. Evitar que "Ah, precisamos fazer X" se perca ou cause ansiedade futura.
2. Identificar se X é um Projeto (vários passos) ou Ação (passo único).
3. Obter 100% de clareza do Próximo Passo Físico.

## O que o agente deve fazer
- Para cada item no inbox bruto, faça a pergunta de clarificação do GTD: "É acionável?"
- Se NÃO: jogue fora, guarde para referência ou anote na lista Someday/Maybe.
- Se SIM: Demora menos de 2 minutos? (Se sim, instrua o usuário a fazer na hora e não processar).
- Se SIM (> 2 min): É um passo ou vários? Se for vários -> Crie o projeto no ClickUp; identifique a PRÓXIMA AÇÃO imediata e atribua.

## Saídas
- Item bruto processado.
- (Se for projeto) Card inicial gerado e aguardando @singer-proj para 'Shaping'.
- Se for tarefa isolada -> Ação atribuída a alguém com verbo físico (Ex: "Ligar para...", "Esboçar a...").
