# Checklist: Controle de Qualidade de Briefing

**Origem do Briefing:** ___ | **Revisor (@singer-proj):** ___

## 1. Viabilidade & Escopo
- [ ] O problema/oportunidade é real e claro?
- [ ] O critério de sucesso é mensurável e inequívoco?
- [ ] O appetite (tempo de execução aceitável) está definido?
- [ ] O escopo (o que está DENTRO) é viável dentro do appetite?
- [ ] O não-escopo (o que está FORA) está explícito para evitar expansão?

## 2. Riscos & Dependências ("Buracos")
- [ ] Identificamos as partes desconhecidas ou não resolvidas do projeto?
- [ ] Existe plano para resolver esses "buracos" antes da execução?
- [ ] Dependemos de aprovação de terceiros incontroláveis?

## 3. Alocação
- [ ] Os agentes/pessoas necessárias têm capacidade para executar?
- [ ] Foi mapeado quem decide, quem executa e quem apenas acompanha?

**BRIEF DE PROJETO APROVADO PARA PITCH: [ ] Sim**
