# Architecture - Virals Marketing Squad

## Design Philosophy
A "Máquina de Viralidade" (Virality Machine) is designed to transform raw concepts into mass attention and scalable revenue. It follows a modular architecture where content creation, distribution, and paid amplification are decoupled but orchestrated by a central strategy.

## Squad Tiers
- **Tier 0 (Orchestration):** `@marketing-chief` - Routes requests and enforces quality.
- **Tier 1 (Core Experts):** `@mrbeast-mk` (Retention), `@garyvee-mk` (Distribution), `@ladeira` (Copy), `@georgi` (VSL), `@perry-marshall` (DR Traffic), `@ezra-firestone` (Brand Traffic).
- **Tier 2 (Support):** `@fishkin-mk` (SEO & Audience Research).

## Workflow Engines
1. **Viral Content Pipeline (WF-001):** The primary engine for content-led growth.
2. **DR Campaign Lifecycle:** Direct Response execution loop.
3. **Brand Funnel Lifecycle:** Long-term audience building engine.

## Data Flow
`Market Trends -> [MrBeast] -> [GaryVee] -> [Ezra/Perry] -> [ROI Data] -> [Marketing Chief]`

## Integration Standards
- **Tasks:** Mandatory HO-TP-001 (8-field) compliance.
- **Agents:** AIOX 6-level loader structure with local file resolution.
- **Workflows:** Phased execution with mandatory human checkpoints.
